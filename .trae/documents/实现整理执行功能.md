# 实现整理执行功能

## 概述
识别功能已完成（TMDB 匹配 → 分类 → 生成目标路径），现在需要补上"最后一公里"：用户确认识别结果后，一键执行整理（创建目录 → 移动/复制 → 重命名）。

## 当前状态
- ✅ 识别流程完整：`organize_service.recognize_file()` 返回 TMDB 信息 + `target_path`
- ✅ 文件操作能力完备：`file_service` 已有 `move_files`、`copy_files`、`rename_file`、`create_folder`
- ✅ 配置齐全：`organize_mode`（move/copy）、`source_path`、`media_library_path` 已存储在 DB
- ❌ 缺少 execute API 和 service 方法
- ❌ 前端识别弹窗没有"执行整理"按钮

## 实施计划

### 1. 后端：organize_service.py 添加 `execute_organize` 方法

**文件**：`d:\OneFive\backend\src\onefive\services\organize_service.py`

新增 `execute_organize(file_info, target_path, organize_mode)` 方法：
1. 解析 `target_path.dir`，按 `/` 拆分为目录层级
2. 从根目录开始，逐级检查/创建目录（调用 `file_service.create_folder`）
3. 获取最终目标目录的 `cid`
4. 根据 `organize_mode` 调用 `file_service.move_files` 或 `copy_files`
5. 如果 `target_path.filename` 与原文件名不同，调用 `file_service.rename_file` 重命名
6. 返回执行结果（成功/失败 + 操作详情）

关键逻辑：
- 目录路径如 `四喜 (2025) {tmdb=273131}` 需要在媒体库路径下创建
- 如果目录已存在则跳过创建，直接获取其 `cid`
- 文件夹整理模式：移动整个文件夹，不重命名内部文件
- 文件整理模式：移动/复制单个文件，然后重命名

### 2. 后端：organize.py 添加 execute API

**文件**：`d:\OneFive\backend\src\onefive\api\organize.py`

新增 `POST /api/organize/execute` 接口：
```python
class ExecuteRequest(BaseModel):
    file_id: str
    file_name: str
    is_dir: bool = False
    target_path: dict  # {"dir": "...", "filename": "..."}
    organize_mode: str = "move"  # "move" 或 "copy"
```

流程：调用 `organize_service.execute_organize()`，返回操作结果。

### 3. 前端：organize.ts 添加 execute API 方法

**文件**：`d:\OneFive\frontend\src\api\organize.ts`

新增：
```typescript
execute(req: ExecuteRequest): Promise<any> {
  return api.post('/organize/execute', req)
}
```

### 4. 前端：RecognizeModal.vue 添加执行按钮

**文件**：`d:\OneFive\frontend\src\components\RecognizeModal.vue`

- 识别结果展示后，在弹窗底部添加"执行整理"按钮
- 按钮点击后调用 `organizeApi.execute()`，传入文件信息 + target_path + organize_mode
- 执行中显示 loading 状态
- 执行成功显示成功提示，关闭弹窗
- 执行失败显示错误信息
- 从设置中读取 `organize_mode` 传入（或让后端自动从 DB 读取）

### 5. 整理流程图

```
用户点击"识别" → 显示识别结果（TMDB信息 + 目标路径）
                        ↓
              用户点击"执行整理"
                        ↓
              后端 execute API
                        ↓
         读取 organize_mode（move/copy）
                        ↓
         解析 target_path.dir，逐级创建目录
                        ↓
         move_files 或 copy_files 到目标目录
                        ↓
         如果文件名需要改变，rename_file
                        ↓
         返回执行结果 → 前端显示成功/失败
```

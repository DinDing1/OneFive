# 通知模块重构计划

## 当前问题

1. **Bot 模式网络问题**：连接超时，可能是代理未正确传入或 API 凭据问题
2. **通知目标理解**：Bot 模式和用户模式都需要 `tg_notify_chat` 指定发送目标
3. **管理员 ID**：两种模式都需要，已实现但需确认
4. **用户模式登录体验差**：需要手动输入 Session 字符串，改为手机号+验证码

## 改动计划

### 1. 修复 Bot 模式网络问题

**文件**：`d:\OneFive\backend\src\onefive\notification\telegram\channel.py`

- `_ensure_client()` 中 Bot 模式：添加 `await self._client.connect()` 再 `start()`，避免超时
- 添加连接超时处理：`timeout=60`，`connection_retries=3`
- 添加 `try/except` 捕获具体错误，返回有意义的错误信息
- 代理传入前日志打印确认

### 2. 用户模式：手机号+验证码登录

**文件**：`d:\OneFive\backend\src\onefive\notification\telegram\channel.py`

新增方法：
- `async def send_code(phone, api_id, api_hash) -> dict` — 发送验证码
  - 创建临时 `TelegramClient(StringSession(''), api_id, api_hash, proxy=...)`
  - 调用 `client.send_code_request(phone)`
  - 保存 `_login_client` 和 `_login_phone` 到实例（5 分钟超时）
  - 返回 `{success, message}`

- `async def sign_in(phone, code, password=None) -> dict` — 验证码登录
  - 用 `_login_client.sign_in(phone, code)` 或带 2FA 密码
  - 成功后 `StringSession.save()` 获取 session_string
  - 自动保存到 DB：`tg_session_string`、`tg_api_id`、`tg_api_hash`
  - 返回 `{success, message}`

- `async def check_login() -> dict` — 检查当前 session 是否有效
  - 用已保存的 session 连接，检查 `is_user_authorized()`
  - 返回 `{logged_in: bool, message: str}`

**文件**：`d:\OneFive\backend\src\onefive\api\notification.py`

新增 3 个端点：
- `POST /api/notification/telegram/send-code` — 发送验证码
- `POST /api/notification/telegram/sign-in` — 验证码登录
- `POST /api/notification/telegram/check-login` — 检查登录状态

### 3. 前端：Settings.vue 重构通知设置 UI

**文件**：`d:\OneFive\frontend\src\views\Settings.vue`

将当前的通知卡片改为更清晰的结构：

```
通知设置
├── 启用通知 [开关]
├── 通知目标 [输入框] — 两种模式都需要
├── Bot 模式

├── 用户模式

│   ├── 手机号 [输入框] + [发送验证码] 按钮
│   ├── 验证码 [输入框] + [登录] 按钮
│   └── 状态：已登录 / 未登录
├── 代理
│   ├── 启用代理 [开关]
│   └── 代理地址 [输入框]
├── 权限
│   └── 管理员 ID [输入框]
```

新增：

**文件**：`d:\OneFive\frontend\src\api\notification.ts`

新增：

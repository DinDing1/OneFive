# 媒体识别整理功能实现计划

## 概述

在 OneFive 后端用 Python 重写媒体识别和整理功能，基于 p115client 调用 115 云盘 API，集成 TMDB 进行媒体文件识别，支持分类、重命名、移动/复制到媒体库目录。

## 技术选型

- **后端框架**：FastAPI（现有）
- **115 云盘**：p115client（现有）
- **TMDB API**：Python requests 直接调用
- **配置存储**：SQLite setting 表（现有）
- **前端**：Vue 3 + Element Plus（现有）

## 功能模块

### 1. TMDB 识别模块 (`services/tmdb_service.py`)

**职责**：调用 TMDB API 搜索和获取媒体详情

**核心方法**：
- `search_media(query, media_type, year)` - 搜索电影/电视剧
- `get_movie_details(tmdb_id)` - 获取电影详情
- `get_tv_details(tmdb_id)` - 获取电视剧详情
- `get_tv_season(tmdb_id, season)` - 获取某季信息

**配置项**（存储在 setting 表）：
- `tmdb_api_key` - TMDB API Key
- `tmdb_api_url` - API 代理地址（可选，默认 https://api.themoviedb.org/3）
- `tmdb_language` - 语言（默认 zh-CN）

**智能匹配算法**：
- 完全匹配标题：100分
- 标题包含查询词：80分
- 查询词包含标题：60分
- 词元相似度：最高40分
- 年份匹配：+30分
- 评分加成：评分 x 2

### 2. 文件名解析模块 (`services/file_info_service.py`)

**职责**：从文件名提取媒体信息

**核心方法**：
- `extract_key_info(filename)` - 提取标题、年份、季集、媒体类型
- `extract_tech_info(filename)` - 提取分辨率、编码、音频、发布组
- `clean_query(filename)` - 清理文件名用于搜索
- `extract_tmdb_id(filename)` - 提取文件名中的 TMDB ID

### 3. 分类模块 (`services/classify_service.py`)

**职责**：根据 TMDB 信息自动分类，采用优先级匹配，先匹配的规则优先生效

**匹配逻辑**：
1. 如果规则指定了类型ID，媒体必须包含至少一个指定类型
2. 如果规则指定了原产国，媒体必须包含至少一个指定国家
3. 所有指定的条件都必须满足（AND 逻辑）
4. 按规则顺序匹配，返回第一个匹配的分类

**默认分类规则（按优先级排序）**：

电影分类：
| 优先级 | 分类 | 条件 |
|--------|------|------|
| 1 | 电影/动画电影 | 类型ID包含 16（动画） |
| 2 | 电影/国产电影 | 原产国为 CN、TW、HK |
| 3 | 电影/日韩电影 | 原产国为 JP、KP、KR |
| 4 | 电影/欧美电影 | 其他所有电影 |

电视剧分类：
| 优先级 | 分类 | 条件 |
|--------|------|------|
| 1 | 剧集/国产动漫 | 类型ID包含 16 + 原产国为 CN、TW、HK |
| 2 | 剧集/日本番剧 | 类型ID包含 16 + 原产国为 JP |
| 3 | 剧集/欧美动漫 | 类型ID包含 16 + 原产国为 US、FR、GB、DE、ES、IT、NL、PT、RU、UK |
| 4 | 其他/纪录片 | 类型ID包含 99 |
| 5 | 其他/综艺 | 类型ID包含 10764（真人秀）或 10767（谈话） |
| 6 | 剧集/儿童 | 类型ID包含 10762 |
| 7 | 剧集/国产剧 | 原产国为 CN、TW、HK |
| 8 | 剧集/日韩剧 | 原产国为 JP、KP、KR、TH、IN、SG |
| 9 | 剧集/欧美剧 | 其他所有剧集 |

### 4. 重命名模块 (`services/rename_service.py`)

**职责**：根据模板生成目标文件名

**默认模板**：

电影：
```
{{title}} ({{year}}) {tmdb={{tmdbid}}}/{{title}}.{{year}}.{{videoFormat}}.{{edition}}.{{videoCodec}}.{{audioCodec}}.{{webSource}}-{{releaseGroup}}{{fileExt}}
```

电视剧：
```
{{title}} ({{year}}) {tmdb={{tmdbid}}}/Season {{season}}/{{title}}.{{seasonYear}}.{{SXXEXX}}.{{videoFormat}}.{{edition}}.{{videoCodec}}.{{audioCodec}}.{{webSource}}-{{releaseGroup}}{{fileExt}}
```

**配置项**：
- `movie_template` - 电影命名模板
- `tv_template` - 电视剧命名模板

### 5. 整理执行模块 (`services/organize_service.py`)

**职责**：执行实际的文件整理操作

**核心方法**：
- `recognize_file(file_id)` - 识别单个文件/文件夹
- `organize_single(file_id, action)` - 整理单个文件
- `organize_batch(file_ids, action)` - 批量整理
- `ensure_path(path)` - 确保目标目录存在（递归创建）
- `handle_subtitles(video_path, target_path)` - 处理关联字幕

**整理流程**：
1. 识别文件（调用 tmdb_service + file_info_service）
2. 生成目标路径（调用 classify_service + rename_service）
3. 确保目标目录存在（调用 115 API 创建目录）
4. 移动或复制文件
5. 重命名文件
6. 处理关联字幕文件
7. 记录整理结果

### 6. API 路由 (`api/organize.py`)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/organize/recognize` | 识别文件 |
| POST | `/api/organize/execute` | 执行整理 |
| GET | `/api/organize/settings` | 获取整理配置 |
| PUT | `/api/organize/settings` | 更新整理配置 |

### 7. 前端页面 (`views/Organize.vue`)

**功能**：
- 文件浏览器（复用现有 Files.vue 组件）
- 识别结果预览（显示 TMDB 信息、建议路径）
- 整理确认对话框（选择移动/复制）
- 设置页面（TMDB 配置、分类规则、命名模板）
